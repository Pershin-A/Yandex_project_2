# Прогнозирование вероятности покупки клиентами

## Описание проекта

Интернет-магазин собирает историю покупателей, проводит рассылки предложений и планирует будущие продажи. Этот проект предназначен для выделения пользователей, которые готовы совершить покупку в ближайшее время.

**Цель:** Предсказать вероятность покупки клиентом в течение 90 дней.

### Задачи
- Изучить и проанализировать данные
- Разработать полезные признаки на основе истории покупок и рассылок
- Создать модель машинного обучения для классификации пользователей
- Улучшить модель и максимизировать метрику ROC-AUC
- Провести тестирование на отложенной выборке

## Описание данных

### apparel-purchases
История покупок клиентов по товарам и датам.
- `client_id` - идентификатор клиента
- `quantity` - количество единиц товара
- `price` - цена товара
- `category_ids` - идентификаторы категорий товара (вложенные)
- `date` - дата покупки
- `message_id` - идентификатор сообщения из рассылки

### apparel-messages
Записи о рассылках и действиях клиентов с сообщениями.
- `bulk_campaign_id` - идентификатор рассылки
- `client_id` - идентификатор клиента
- `message_id` - идентификатор сообщения
- `event` - действие (отправлено, открыто, покупка и т.д.)
- `channel` - канал рассылки (email, mobile_push и т.д.)
- `date` - дата действия
- `created_at` - дата-время полностью

### apparel-target_binary
Целевой признак.
- `client_id` - идентификатор клиента
- `target` - совершил ли клиент покупку в течение 90 дней (0 или 1)

### full_campaign_daily_event
Агрегированная статистика по рассылкам по дням и типам событий.
- `date` - дата
- `bulk_campaign_id` - идентификатор рассылки
- `count_event*` - количество каждого события
- `nunique_event*` - количество уникальных клиентов по событиям

### full_campaign_daily_event_channel
Агрегированная статистика по рассылкам, каналам и событиям.
- `date` - дата
- `bulk_campaign_id` - идентификатор рассылки
- `count_event*_channel*` - количество событий по каналам
- `nunique_event*_channel*` - количество уникальных клиентов

### Запуск анализа

1. **Подготовка данных** - откройте и запустите `Формировние данных.ipynb`:
   - Загрузка исходных данных

   - Сохранение признаков в CSV

2. **Обучение модели** - откройте и запустите `Обучение модели.ipynb`:
   - Загрузка сформированных признаков
   - Разделение на обучающую и тестовую выборки
   - Обучение и оптимизация моделей LightGBM и CatBoost
   - Оценка качества на тестовой выборке

### Результаты

После выполнения ноутбуков получится:
- Обученная модель CatBoost для прогнозирования вероятности покупки
- Значение метрики ROC-AUC на тестовой выборке (~0.704)
- Предсказания для всех клиентов в виде вероятностей

## Архитектура модели

Финальный конвейер обработки:

1. **Предобработка данных** (ColumnTransformer):
   - Числовые признаки: заполнение пропусков нулями или единицами
   - Категориальные признаки (common_channel): кодирование в числовые значения

2. **Модель CatBoost**:
   - 400 итераций
   - Глубина дерева: 3
   - Learning rate: ~0.034
   - L2 регуляризация: ~34.3
   - Метрика: ROC-AUC

## Ключевые особенности

- **Обработка пропусков**: Клиенты без информации о рассылках заполняются нулями в соответствующих признаках
- **Балансировка классов**: Используется StratifiedKFold для сохранения пропорций классов при кроссвалидации
- **Оптимизация гиперпараметров**: Optuna проводит поиск лучших параметров для обеих моделей
- **Параллельная обработка**: Использование joblib и n_jobs=-1 для ускорения вычислений
